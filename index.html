The Challenge

New candidates face complex onboarding journeys from offer acceptance through day one. Each touchpoint is an opportunity to create a seamless experience.

The challenge? Moving quickly from pain points to interactive solutions that business partners can experience firsthand.

Traditional development takes months. What if we could compress that to hours?

Process Mapping

Starting with the User Journey

We began by mapping the complete candidate experience in Figma—documenting every touchpoint, decision point, and friction area.

From Discovery to Prototype

What We Discovered

  • Candidates needed a single source of truth
  • Status tracking was unclear and fragmented
  • Multiple systems caused confusion
  • No centralized resource hub
  • Pre-boarding tasks were scattered

Solution

  • Requirements to prototype in hours
  • Interactive, mobile-responsive application
  • Built-in features and status tracking
  • Real-time iteration with stakeholders
  • Immediate feedback incorporation

The Transformation

FROM

  • Static diagrams
  • Requires interpretation
  • Abstract concepts
  • Weeks of iteration
→

TO

  • Working prototypes
  • Self-explanatory UX
  • Tangible experience
  • Hours of iteration

The Result: Hiring Hub

A functional prototype delivered in real-time conversations with stakeholders

Key Features

5-step visual progress tracking, interactive task cards, file upload for badge photos, direct system links, parking info and pre-work checklists

What Made It Work

Mobile-first responsive design, Lilly brand alignment, intuitive navigation, built-in help and support

Stakeholder Impact

Immediate interaction with solution, feedback loops compressed from weeks to minutes, hands-on testing, rapid iteration

Business Value

Faster decision-making, reduced development risk, validated concepts early, compressed timeline by 90%
